Đây là toàn bộ API cho trang Lịch học (/my-space/calendar) của Student/Phụ huynh:

📋 API Lịch học — Student / Phụ huynh
Base URL: https://<domain>
Auth: Cookie session (hoặc token — tất cả endpoint đều yêu cầu đăng nhập)
Lưu ý: Nếu tài khoản là Phụ huynh, API tự động trả về dữ liệu của tất cả học viên con được liên kết. Dùng query ?studentId=<id> để lọc theo từng con.

1. Xác định loại tài khoản
GET /api/my-space/user-type
Response:
{
 "userType": "student" | "staff" | null
}
Phụ huynh sẽ trả về "student" vì hệ thống coi phụ huynh là một dạng học viên context.

7. Ghi nhận học viên vào lớp online (Check-in)
POST /api/my-space/calendar/student/session/:classSessionId/online-click?studentId=<uuid>
Response:
{ "onlineClickedAt": "2025-06-02T08:01:00Z" }

8. Ghi nhận học viên kết thúc lớp online (Check-out)
POST /api/my-space/calendar/student/session/:classSessionId/online-end?studentId=<uuid>
Response:
{ "onlineEndedAt": "2025-06-02T10:05:00Z" }
⚠️ Phải gọi API check-in (online-click) trước, nếu chưa sẽ trả lỗi 400.

9. Ghi nhận lượt làm bài kiểm tra trong buổi test
POST /api/my-space/test-content-attempt
Request body:
{
 "testSessionId": "uuid",
 "contentId": "uuid",
 "contentType": "exam" | "assignment",
 "studentId": "uuid"   // tùy chọn, Phụ huynh dùng để chỉ định con
}
Response (còn lượt):
{ "allowed": true, "attemptsUsed": 1, "maxAttempts": 3 }
Response (hết lượt):
{ "allowed": false, "attemptsUsed": 3, "maxAttempts": 3 }

Lỗi chung
HTTP Code
Ý nghĩa
401
Chưa đăng nhập
403
Tài khoản không phải học viên / phụ huynh
404
Không tìm thấy buổi học
400
Dữ liệu không hợp lệ (vd: chưa check-in mà đã check-out)
500
Lỗi server


Hệ thống đã có đầy đủ 9 API cho tính năng Lịch học của Student/Phụ huynh. Bạn có thể chuyển tài liệu này cho team mobile app sử dụng trực tiếp.
